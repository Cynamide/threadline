# CLI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `@threadline/cli` as the workspace command layer for initialization, validation, handoff scanning, and hook installation.

**Architecture:** Keep the CLI as a thin orchestrator over package-local detectors, generators, and the AST guard. `init` discovers repo shape and writes `.threadline/` config artifacts, `validate` passes staged or full-file inputs to the validator, `scan-handoffs` turns parsed handoffs into tracker-ready records, and `install-hooks` writes the pre-push hook.

**Tech Stack:** TypeScript, Node.js filesystem APIs, YAML generation, git command execution, pnpm workspace, and fixture repos for integration tests.

## Global Constraints

- Creates the repo-local configuration under `.threadline/`
- Runs the AST guard against staged files or the full source tree and reports violations in text or JSON
- Extracts all `handoff()` calls and returns structured records for issue tracker export
- Writes or updates the local pre-push hook so validation runs before code leaves the machine
- Detects framework: Next.js, Vite, CRA, Remix, or custom
- Detects styling: Tailwind, styled-components, Emotion, CSS modules, or plain CSS
- Detects design system: shadcn, MUI, Ant Design, Radix, or custom
- Generated files: `.threadline/config.yaml`, `.threadline/boundaries.md`, `.threadline/design-system.md`, `.threadline/skill.md`

- [ ] **Step 3: Implement the detector and generator shells**

```ts
export function detectFramework(input: DetectFrameworkInput): FrameworkKind {
  if (input.packageJson.dependencies?.next) return 'nextjs';
  return 'custom';
}

export function detectStylingStrategy(input: DetectStylingInput): StylingKind {
  if (input.packageJson.dependencies?.tailwindcss) return 'tailwind';
  if (input.packageJson.dependencies?.['styled-components']) return 'styled-components';
  return 'plain-css';
}
```
