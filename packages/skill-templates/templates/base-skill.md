# Base Skill

Read the repo's docs, config, and nearby code before editing.
Use the repo vocabulary: skill, plan, agent brief, and boundary rule.
If the change spans multiple files or concepts, write a plan before editing.
Stay inside the boundary rule. Do not edit files outside the allowed surface.
Use `handoff()` for work that should remain engineer-owned.
Run the relevant validation before you say the work is done.
When the repo needs tracker output, keep the agent brief portable and let the adapter handle export details.
