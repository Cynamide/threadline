# Context Map

This repo uses multiple local contexts. Each package context is a glossary for the terms that matter in that part of the system.

## Package contexts

- [`packages/runtime/CONTEXT.md`](./packages/runtime/CONTEXT.md) - runtime vocabulary
- [`packages/cli/CONTEXT.md`](./packages/cli/CONTEXT.md) - CLI vocabulary
- [`packages/ast-guard/CONTEXT.md`](./packages/ast-guard/CONTEXT.md) - validation vocabulary
- [`packages/skill-templates/CONTEXT.md`](./packages/skill-templates/CONTEXT.md) - agent-instruction vocabulary

## Reading order

1. Read this map first
2. Read the package context for the area you are changing
3. Read relevant ADRs in `docs/adr/`
